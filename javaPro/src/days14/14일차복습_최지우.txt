1. "싱글톤 클래스"를 구현하는 방법에 대해서 설명하세요...
	- 싱글톤이란, 특정 클래스의 인스턴스(객체)를 하나만 생성하고 그 인스턴스를 전역적으로 접근할 수 있도록 하는 것이고, 구현하는 방법은 몇가지가 있지만(지연초기화, 이중잠금 등) 결국 어떤 인스턴스가 생성되지 않았으면 생성하되 생성되어있는 상태면 더 이상 생성하지 않게 구현한다.

2. instanceof 연산자에 대해서 설명하세요. 
[객체이름] instanceof [클래스이름] 의 형식으로 사용하며, true / false를 반환한다. 
매개변수로 전달된 객체가 어떤 클래스인지 확인하는 용도로 사용하며, 만약 여러 클래스중 어떤 클래스의 객체인지 확인하고 싶은데 클래스끼리 상속관계인 경우 사용에 주의가 필요하다.

3. [매개변수 다형성]에 대해서 설명하세요 . 
자동 업캐스팅으로 인해 가능한 것으로, 매개변수에 부모클래스나 더 상위 클래스 등을 넣으면 자식클래스들은 알아서 자동 업캐스팅 되며 본인의 기능을 하기때문에 다형성이라고 한다.

4. [] 대괄호 안에 있다. 없다 골라서 적으세요.
  1) 메서드에 static과 abstract를 함께 사용할 수 [  없다 ]. 
  2) 클래스에 static과 final을 동시에 사용할 수 [ 없다 ]. 
  3) abstract메서드의 접근 제어자가 private일  수 [ 없다 ]. 
  4) 메서드에 private과 final을 같이 사용할 필요 [  ].

5. 클래스(OOP)의 특징에 대해서 상세히 설명하세요. 
객체지향 프로그래밍(Object-Oriented Programming, OOP), 추상화, 캡슐화, 상속, 다형성

6. abstract 키워드에 대해서 설명하세요. 
자바에서 추상화를 위해 사용되는 키워드로, 추상 클래스와 추상 메서드를 정의할 때 사용한다.
추상 클래스는 객체를 직접 생성할 수 없다. 상속을 통해 자식 클래스에서 해당 추상 클래스를 구체적으로 구현해야 합니다.

7. 인터페이스(interface) 에 대해서 상세히 설명하세요.
인터페이스는 참조형에 속하며, 일종의 추상클래스이다. 일반 메서드 선언을 하지 않고 추상 메서드만 존재한다. 
필드를 선언할 수 없고 상수만 선언할수있다. 다른 클래스를 작성하는 데 도움을 줄 목적의 클래스이다.
